## A

**Acquire memory ordering** — Семантика упорядочения памяти, гарантирующая, что все последующие операции чтения/записи не будут переупорядочены до текущей операции.

**Address Sanitizer (ASan)** — Инструмент для детекции ошибок работы с памятью (use-after-free, buffer overflow, memory leaks).

**Atomic operation** — Неделимая операция, выполняющаяся как единое целое без возможности наблюдения промежуточных состояний.

---

## B

**Backpressure** — Механизм регулирования потока данных, когда медленный потребитель естественным образом замедляет производителя.

**Baseline scenario** — Базовый тестовый сценарий с равномерной нагрузкой для проверки основной функциональности.

**Benchmark** — Измерение производительности отдельного компонента или операции.

**Bottleneck** — Узкое место в системе, ограничивающее общую производительность.

**Burst traffic** — Всплеск нагрузки, значительное кратковременное увеличение интенсивности трафика.

**Busy-waiting (spin-waiting)** — Активное ожидание в цикле без передачи управления операционной системе, обменивает CPU usage на низкую latency.

---

## C

**Cache line** — Минимальная единица кеша процессора, обычно 64 байта на x86-64.

**Cache-line alignment** — Выравнивание данных по границе cache line для предотвращения false sharing.

**Compare-and-exchange (CAS)** — Атомарная операция сравнения и обмена, основа многих lock-free алгоритмов.

**Contention** — Конкуренция потоков за доступ к общему ресурсу.

---

## D

**Data race** — Ситуация, когда два или более потока одновременно обращаются к одной ячейке памяти, и хотя бы одно обращение — запись, без синхронизации.

**Deterministic routing** — Детерминированная маршрутизация, когда путь сообщения однозначно определяется его типом.

**Discrete distribution** — Дискретное вероятностное распределение для генерации случайных типов сообщений.

---

## E

**End-to-end latency** — Задержка от создания сообщения производителем до финальной обработки стратегией.

---

## F

**False sharing** — Ситуация, когда разные потоки работают с разными переменными, находящимися в одной cache line, вызывая неэффективность.

**Fetch-and-add** — Атомарная операция чтения и инкремента значения.

**FIFO (First-In-First-Out)** — Принцип очереди, где первый пришедший элемент обрабатывается первым.

---

## G

**Graceful shutdown** — Корректное завершение работы с обработкой всех оставшихся сообщений.

---

## H

**Happens-before** — Отношение упорядочения в модели памяти C++, гарантирующее видимость операций между потоками.

**High-Frequency Trading (HFT)** — Высокочастотная торговля, требующая микросекундных задержек.

**Hotspot** — Неравномерная нагрузка, когда большая часть трафика приходится на один элемент.

---

## L

**Latency** — Задержка, время от начала до завершения операции.

**Lock-free** — Свойство алгоритма, гарантирующее, что хотя бы один поток делает progress в конечное время без использования блокировок.

---

## M

**Memory ordering** — Гарантии порядка выполнения операций с памятью в многопоточной среде.

**Message loss** — Потеря сообщений в системе, недопустимая в данном проекте.

**MPSC (Multi-Producer-Single-Consumer)** — Очередь с множеством производителей и одним потребителем.

---

## N

**NUMA (Non-Uniform Memory Access)** — Архитектура с неравномерным временем доступа к разным областям памяти.

---

## O

**Ordering violation** — Нарушение порядка доставки сообщений от одного производителя.

---

## P

**Percentile (p50, p90, p99)** — Процентиль распределения, значение, ниже которого находится указанный процент данных.

**POD (Plain Old Data)** — Простой тип данных без виртуальных функций и сложных конструкторов.

**Producer** — Производитель, компонент, генерирующий сообщения.

**Processor** — Процессор, компонент, имитирующий обработку сообщений.

---

## R

**Release memory ordering** — Семантика упорядочения, гарантирующая, что все предыдущие операции чтения/записи завершились до текущей операции.

**Relaxed memory ordering** — Минимальные гарантии упорядочения, только атомарность операции.

**Ring buffer** — Кольцевой буфер, структура данных фиксированного размера с циклическими индексами.

**Round-robin** — Циклическое распределение нагрузки между несколькими обработчиками.

**Router** — Маршрутизатор, компонент, направляющий сообщения по заданным правилам.

---

## S

**Sampling** — Выборка, сбор статистики не для всех элементов, а для репрезентативной выборки.

**Sequence number** — Порядковый номер сообщения от производителя, монотонно возрастающий.

**SPSC (Single-Producer-Single-Consumer)** — Очередь с одним производителем и одним потребителем.

**Stage1 Router** — Маршрутизатор первой стадии (производители → процессоры).

**Stage2 Router** — Маршрутизатор второй стадии (процессоры → стратегии).

**Strategy** — Стратегия, финальный потребитель сообщений с валидацией порядка.

---

## T

**Thread Sanitizer (TSan)** — Инструмент для детекции data races в многопоточных программах.

**Throughput** — Пропускная способность, количество обработанных элементов в единицу времени.

**Timestamp** — Временная метка, момент времени в наносекундах.

**Trivially copyable** — Свойство типа, допускающее побайтовое копирование (memcpy-safe).

---

## U

**UB (Undefined Behavior)** — Неопределенное поведение, ситуация, не определенная стандартом языка.

**UB Sanitizer (UBSan)** — Инструмент для детекции undefined behavior.

---

## W

**Wait-free** — Более строгое свойство, чем lock-free: каждый поток гарантированно делает progress за конечное время.
